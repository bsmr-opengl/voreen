VoreenVE - Getting Started

VoreenVE (Voreen Visualization Environment) is a tool to construct processor networks for Voreen with an easy to use graphical user interface. It allows to create complex rendering networks by simply dragging the desired processors into the workspace with the mouse and linking them by drawing the required connections. The networks can be tested immediately to visualize volume data sets.

Main Window

The following window appears when starting the VoreenVE application:


  • The green numbers indicate the main components:
    1. The network editor shows the current processor network and allows the user to modify it interactively.
    2. The visualization window shows the result produced by the current network (you need to press the evaluate button to start rendering).
    3. The processors list shows all available processors. These can be used to construct networks.
    4. The properties allow configuration of the selected processor.

Modes

VoreenVE can be used in network mode and visualization mode. Rendering networks are constructed in network mode. Switching to visualization mode hides the network editor and displays a larger visualization window. While the layout of the rendering network cannot be changed in visualization mode, it is still possible to modify properties of the processors in the network.

Toolbar

The picture below shows the toolbar entries and their function.


  • New Workspace: Create a new empty workspace.
  • Open Workspace: Load an existing workspace.
  • Save Workspace: Save the current workspace.
  • Open Network: Load an existing network, replacing the current network in the workspace.
  • Open Data Set: Load a volume data set (See Loading data sets).
  • Evaluate Network: Evaluate the network and start rendering. This command is needed when new processors were added to the network or connections were changed.
  • Processors: Hide/show the processor list (only available in network mode).
  • Properties: Hide/show the properties.
  • Volumes: Hide/show information about the currently loaded volume data sets.
  • Debug Console: Hide/show a debug console containing log messages.
  • Texture Container: Hide/show the texture container, which shows intermediate rendering results, useful for debugging.
  • Network Mode: Switch to network mode.
  • Visualization Mode: Switch to visualization mode.
  • Camera Orientation: Allows finer control of the camera position, as well as loading/saving of the camera state.
  • Snapshot: Save the content of the visualization window into an image file.
  • Animation: Create animations using camera paths and save these animations as videos.

Networks and Workspaces

VoreenVE can save and load networks and workspaces. Networks contain the current rendering network, including all properties. Workspaces contain additional information such as the loaded volume data sets.

Creating Render Networks

In the network editor the graphical representation of the current network is shown. In order to build a rendering network, you can simply add new processors by dragging them from the processor list into the network editor. Processor can be connected via their different ports, however the port types have to match (different types are identifiable by their colors).

Ports are connected by pressing the left mouse button over an outport (located on the lower side of a processor) and dragging the appearing arrow to an inport (on the upper side of a processor). The arrow indicates whether a connection to the inport at the current mouse position is possible or not by changing its color to green respective red. Releasing the mouse button over a valid inport (green arrow) creates a connection.

Working with Networks

In the following described how a network can be modified by the user.

  • An item it selected by simply clicking it. Multiple items can be selected by holding the CTRL key and selecting another one or by spanning a rectangle over the items using the left mouse button.
  • An item can be moved by clicking on it, moving it to the desired position and releasing the mouse button.
  • Renaming a processor or an aggregation can be done using the context menu.
  • Items can be deleted by selecting them and either using the delete entry from the context menu or just pressing the DEL key.
  • The workspace can be zoomed in and out via the mouse wheel or by using the + and - keys.

After building a network you can start it by evaluating it pressing in the toolbar. If no error occurs, an output image is rendered and displayed in the visualization window.

Visualization Window

After setting up a valid network you can start it by clicking on the toolbar item . The output of the network is then rendered in the visualization window. You can rotate the camera by moving the mouse in the window while pressing the left mouse button. Using the mouse wheel or moving the mouse up and down while pressing the right mouse button, zooms the visualization. Pressing SHIFT and the left button moves the scene.

Processor List

The processor list displays all available processors that can be used to build networks. You can search for processors by typing in the text box at the top.


You can add processors to your network simply by dragging them from the list into the network editor.

Loading Data Sets

To load a data set simply click on "Open Data Set" in the toolbar or in the file-menu. The data set is then added to the volume container. To render the loaded data set select the VolumeSetSource processor in the network and change the VolumeSet property.


Texture Container

The texture container can be used to inspect intermediate rendering results. A left-click enlarges a texture, a second click shrinks it again. With the context menu, alpha-channels and depth buffers can be accessed.
